# TableResearch

## Условие задачи
Создать таблицу, содержащую не менее 40-ка записей (тип – запись с вариантами (объединениями)). Упорядочить данные в ней по возрастанию ключей, двумя алгоритмами сортировки, где ключ – любое невариантное поле, используя: а) саму таблицу, б) массив ключей.
Ввести список студентов, содержащий следующие сведения о них: Фамилия, имя, группа, пол (м/ж), возраст, средний балл за сессию, дата поступления, адрес в зависимости от типа жилья: 
1. Адрес дома: улица, № дома, № квартиры
2. Адрес общежития: № общежития, № комнаты
3. Адрес съемного жилья: улица, № дома, № квартиры, стоимость

Вывести список студентов указанного года поступления, живущих в съемном жилье, стоимостью меньше указанного.
## Исходные данные: 
Взаимодействие с программой происходит через меню-интерфейс. Пользователь вводит числа (0-9) для выбора действий программы.

1, чтобы вывести студентов неотсортированными

2, чтобы добавить студента

3, чтобы импортировать записи из students.txt

4, чтобы удалить студента

5, чтобы просмотреть отсортированную таблицу ключей

6, чтобы просмотреть отсортированную исходную таблицу

7, чтобы просмотреть отсортированную таблицу с использованием таблицы ключей

8, чтобы просмотреть студентов по году поступления, снимающих квартиру стоимостью меньше указанной

9, чтобы анализиривать время выполнения

0, чтобы завершить

При описании имени и фамилии студента вводятся строки длиной до 20 символов. При описании группы студента вводится строка длиной до 7 символов. При описании пола студента вводится m, для мужчин и f, для женщин. При описании возраста студента вводится целое число от 10 до 40. При описании средней оценки студента за сессию вводится вещественное число в интервале от 1.0 до 5.0. При описании даты поступления вводится дата в формате DD-MM-YYYY в интервале от 01-01-1900 до 31-12-2025. При описании адресса, сначала вводится тип жилья: 1 – квартира в собственности, 2 — общежитие, 3 — съёмная квартира; далее для съёмной и собственной квартиры требуется указать улицу (строка до 20 символов), номер дома и квартиры (целые положительные числа), для съёмной квартиры нужно также указать стоимость аренды (целое число от 3000 до 50000); для общежития нужно ввести номер общежития и комнаты в нём (целое положительное число).

<ins>Важно: при использовании русскоязычных символов в строковых полях поведение программы может быть нарушено, использование их не рекомендовано.</ins>
## Выходные данные:
Таблица студентов (для пунктов 1, 6, 7, 8), таблица ключей (для пункта 5), таблица анализа сортировок (для пункта 9). В остальных случаях вывода не происходит. При вводе 0, программа перманентно завершается без каких-либо сообщений.
Задача, реализуемая программой
Различные подходы к сортировке таблиц, исследование эффективности этих подходов.
## Способ обращения к программе
Через терминал (app.exe).
## Возможные аварийные ситуации и ошибки пользователя
Некорректный ввод (использование недопустимого символа, превышение максимально возможного количества символов, неверный формат ввода, ввод значения не в пределах допустимого интервала).
Попытка считывания данных из students.txt, если данный файл отсутствует.
Попытка добавления значения в заполненную таблицу.
